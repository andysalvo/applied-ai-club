# Security Policy

## Supported Scope
This policy applies to all repository content and automation/workflow code in this project.

## Reporting a Vulnerability
Please report vulnerabilities privately.

Preferred path:
- Use GitHub's private vulnerability reporting for this repository (Security tab -> "Report a vulnerability").

Fallback path:
- Contact project maintainers directly through GitHub profile/contact channels.

Please do not open public issues with exploit details.

## What to Include
- A clear description of the issue.
- Steps to reproduce.
- Potential impact.
- Any suggested mitigation.

## Response Expectations
- Initial acknowledgement target: within 3 business days.
- Initial triage target: within 7 business days.
- Resolution timing depends on severity, complexity, and contributor availability.

## Disclosure
After a fix is available (or mitigation is in place), maintainers may coordinate responsible public disclosure.
