# Language Style Guide — AI in Accounting Embeds

## Purpose

Keep language calm, clear, and professor-like. The goal is to help students reason about accounting practice, not to advertise technology.

## Banned Words and Phrases (UI Copy)

Do not use these terms in visible interface copy:

- Explorer
- Dashboard
- Module
- Tool
- Framework
- Interactive

Do not use high-friction instruction copy such as:

- "Click each item to inspect..."
- "Use this interface to explore..."

## Required Section Titles

Use these exact section titles:

- Where AI Appears in Accounting
- What AI Helps With
- Where AI Falls Short
- What to Keep in Mind

## Tone Rules

- Prefer concrete accounting terms over abstract AI language.
- Use short sentences.
- Explain process and limits without hype.
- Keep claims descriptive and bounded.
- Always include where professional judgment remains required.

## Microcopy Rules

Hints should be 3–6 words:

- Section 1: "Choose an area."
- Section 2: "Pick a task."
- Section 3: "Select a misconception."
- Section 4: "Start anywhere."

Control labels:

- Use "What’s true in practice" instead of reveal-jargon labels.
- Use "A common assumption" for misconception framing.
- Use "Return" for close/back controls.

Panel metadata:

- Prefer natural labels: `Selected: Auditing`
- Avoid all-caps system labels.

## Explanation Panel Template

Every selected item should use:

1. Context line (1 short sentence)
2. Explanation body (2–4 short sentences)
3. Boundary line (1 short sentence about standards, documentation, oversight, or judgment)

## Readability Rules

- Keep paragraphs to 1–3 sentences.
- Use bullets only when they speed scanning.
- Keep instructions minimal and put depth in reading panel content.
- Avoid repeating the same sentence structure across all items.
